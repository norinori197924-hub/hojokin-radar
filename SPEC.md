## 1. 概要と目的

補助金レーダーのトップページを、利用者が「自分に使える補助金」を最短で見つけられる検索・一覧体験へ改善する。あわせてアクセス解析(GA4)・AdSense審査に必要な情報開示ページ・独自ガイドコンテンツを整備し、公開後の改善判断と収益化準備をデータと運用基盤に基づいて行えるようにする。

## 2. ターゲットユーザーと主要な導線

- 補助金を探す中小企業・個人事業主・自治体関係者
- 主要導線:トップページ → 検索/絞り込み → 一覧カード → 詳細ページ
- 検索の優先順位:地域 → 目的キーワード → 業種 → 新着 → 締切

#### コラム1 確定稿:よくある不採択理由と対策

補助金は、要件を満たせば必ず受け取れるものではありません。多くの補助金は予算と採択件数に限りがあり、申請内容を審査したうえで採択・不採択が決まります。せっかく時間をかけて申請するなら、落ちやすいパターンをあらかじめ知っておくことが大切です。ここでは、補助金申請でよく指摘される不採択の理由と、その対策を紹介します。

**理由1:公募要領の要件を満たしていない**

最も基本的でありながら、意外と多いのが「そもそも応募要件を満たしていない」ケースです。対象となる事業者の規模(中小企業の定義)、業種、地域、事業内容などの条件は補助金ごとに細かく定められています。また、必要書類の不足や様式の誤り、締切超過といった形式面の不備も、内容を審査される前に不採択となる典型的な原因です。

対策:申請前に公募要領を最初から最後まで読み、応募資格・対象経費・必要書類をチェックリスト化しましょう。とくに「対象となる方」「対象とならない方」の記載は必ず確認してください。不明点は公募元の問い合わせ窓口に事前に確認するのが確実です。

**理由2:事業計画と補助金の目的がずれている**

補助金は、国や自治体が「この方向の取り組みを増やしたい」という政策目的のために設けています。申請する事業の内容がその目的と合っていないと、事業自体がどれだけ立派でも採択されにくくなります。「使えそうな補助金があったから申請する」という発想で、自社の事業を無理に補助金の趣旨に合わせようとすると、計画の一貫性が失われ、審査側にも見抜かれます。

対策:公募要領の冒頭にある「事業の目的」「背景」をよく読み、自社の取り組みがその目的にどう貢献するのかを、事業計画の中で明確に言語化しましょう。目的との接点が薄いと感じたら、その補助金への申請自体を見直すのも判断のひとつです。

**理由3:事業計画の具体性・実現可能性が弱い**

審査では「この計画は本当に実行できるのか」「補助金を出した結果、何がどう良くなるのか」が見られます。市場や顧客の分析が漠然としている、数値目標の根拠が示されていない、実施体制やスケジュールが曖昧、収支計画に無理がある——こうした計画は説得力を欠き、評価が伸びません。

対策:「誰に、何を、どのように提供し、その結果どんな効果が出るのか」を具体的に書きましょう。数値目標には「なぜその数字なのか」の根拠を添えます。自社の強みや実績、実施体制(誰が何を担当するか)も明記し、第三者が読んで実行のイメージが湧く計画に仕上げることが重要です。

**理由4:加点項目・審査項目を意識していない**

多くの補助金には、審査の観点(審査項目)や、満たすと評価が上がる加点項目が公開されています。これらを確認せずに申請書を書くと、審査側が見たいポイントに答えていない申請書になりがちです。

対策:公募要領や関連資料に審査項目・加点項目の記載があれば、申請書の各項目がそれに対応しているかを照らし合わせながら書きましょう。加点の対象となる認定や取り組み(制度によって異なります)が自社で取得可能なものであれば、申請前に準備しておくことも検討に値します。

**理由5:書類の完成度・伝わりやすさが低い**

内容は良くても、書き方で損をしているケースもあります。専門用語の多用、論理の飛躍、誤字脱字、図表のない長文などは、審査員に内容が正しく伝わらない原因になります。審査員は多数の申請書を読むため、短時間で要点が掴める書類が有利です。

対策:結論を先に書く、段落ごとに主張をひとつに絞る、必要に応じて図表で整理する、第三者(社内の別の担当者や支援機関)に読んでもらう——といった基本的な工夫が効果的です。商工会議所や中小企業支援機関などの無料相談を活用するのもよいでしょう。

**まとめ**

不採択の理由は、突き詰めると「要件の確認不足」「目的とのミスマッチ」「計画の説得力不足」の3つに集約されます。いずれも、公募要領を丁寧に読み込み、審査する側の視点に立って申請書を作ることで大きく改善できます。一度不採択になっても、公募が複数回ある補助金では計画を見直して再挑戦できる場合もあります。

※本記事は一般的な情報の整理であり、個別の補助金制度の要件・審査基準はそれぞれ異なります。申請にあたっては、必ず各制度の公式の公募要領をご確認ください。

**ステップ1:公募情報の確認**

補助金には「公募期間」があり、その期間内にしか申請できません。気になる補助金を見つけたら、まず公式ページで公募要領(募集のルールをまとめた文書)を入手し、以下を確認します。

- 応募資格(事業者の規模・業種・地域など)
- 補助対象となる経費・ならない経費
- 補助率と上限額
- 公募期間と締切
- 必要書類

公募期間が短い制度もあるため、締切から逆算して準備のスケジュールを立てることが重要です。

**ステップ2:事業計画の作成・申請書類の準備**

補助金申請の中心となるのが事業計画書です。「何のために、何をして、どんな成果を目指すのか」を、公募要領の様式に沿ってまとめます。あわせて、決算書や登記簿謄本(履歴事項全部証明書)、見積書など、制度ごとに指定された添付書類を揃えます。

書類の準備には想像以上に時間がかかることが多いため、締切間際ではなく早めに着手しましょう。分からない点は、公募元の問い合わせ窓口や、商工会議所・よろず支援拠点などの中小企業支援機関に相談できます。

**ステップ3:申請**

指定された方法で申請します。近年は、国の補助金を中心に電子申請システム(jGrants など)を使うケースが増えています。電子申請には「GビズID」などのアカウントが事前に必要な場合があり、アカウントの発行に時間がかかることがあるため、申請方法は早い段階で確認しておきましょう。

**ステップ4:審査・採択発表**

提出された申請は、公募元による審査を経て、採択・不採択が決まります。審査期間は制度によって異なり、数週間から数か月かかることもあります。採択結果は、通知や公募元サイトでの公表によって知らされます。

**ステップ5:交付申請・交付決定**

ここが初めての方が最も誤解しやすいポイントです。「採択」はまだゴールではありません。採択後に「交付申請」という手続きを行い、「交付決定」を受けて初めて、補助対象となる事業を正式に開始できます。

多くの制度では、交付決定前に発注・契約・支払いをした経費は補助対象になりません。採択の連絡に安心して先に発注してしまい、その分が対象外になる——という失敗は典型例なので、必ず交付決定を待ってから事業を始めましょう(制度により事前着手が認められる場合もありますが、例外的な扱いです)。

**ステップ6:事業の実施**

交付決定の内容に沿って事業を実施します。このとき、経費の支払いを証明する書類(見積書・発注書・納品書・請求書・振込記録など)を漏れなく保管しておくことが極めて重要です。後の実績報告で提出を求められます。

**ステップ7:実績報告・確定検査**

事業が完了したら、期限内に実績報告書を提出します。実施した内容と支出した経費を報告し、公募元の確認(確定検査)を受けて、補助金額が確定します。

**ステップ8:補助金の受け取り**

補助金額の確定後、請求手続きを経て補助金が振り込まれます。ここで押さえておきたいのが、補助金は原則「後払い(精算払い)」だということです。事業にかかる費用はいったん全額自己資金(または融資)で立て替え、事業完了後に補助分が戻ってくる、という流れが基本です。資金繰りの計画は、この前提で立てる必要があります。

また、制度によっては事業完了後も数年間、状況報告や帳簿類の保存義務が続く場合があります。受け取って終わり、ではない点も覚えておきましょう。

**まとめ**

補助金は「見つける → 申請する → 採択される → 交付決定を受ける → 実施する → 報告する → 受け取る」という長い道のりです。特に「交付決定前の発注はNG」「原則後払い」の2点は、知らないと実害につながりやすい重要ポイントです。全体の流れを頭に入れたうえで、締切から逆算して余裕を持って準備を進めましょう。

※本記事は一般的な情報の整理であり、個別の補助金制度の手続き・要件はそれぞれ異なります。申請にあたっては、必ず各制度の公式の公募要領をご確認ください。

#### コラム3 確定稿:補助金の申請前に知っておくべき注意点

補助金は返済不要の心強い制度ですが、「もらえるお金」という側面だけを見て申請すると、思わぬ落とし穴にはまることがあります。ここでは、申請の前に知っておきたい、補助金という制度そのものの性質に関わる注意点をまとめます。申請手続きの流れについては、別のコラム「補助金申請の基本的な流れ」もあわせてご覧ください。

**注意点1:同じ経費に複数の補助金は使えない(二重受給の禁止)**

原則として、同一の事業・同一の経費に対して、国の補助金を重複して受け取ることはできません。たとえば、同じ設備の購入費を2つの補助金それぞれに計上する、といったことは認められていません。複数の制度を活用したい場合は、事業や経費を明確に切り分ける必要があります。重複が発覚した場合、交付決定の取消しや補助金の返還を求められることがあり、悪質なケースでは罰則の対象にもなり得ます。

**注意点2:補助金は課税対象になる**

補助金は「もらって終わり」のお金ではなく、会計上は収入(雑収入等)として扱われ、法人税・所得税の課税対象になります。「補助金で設備を買ったのに、思ったより税負担が増えた」という事態を避けるため、受給する年度の損益への影響をあらかじめ把握しておきましょう。固定資産の取得に補助金を使った場合には「圧縮記帳」という税負担を繰り延べる制度が使える場合もあります。具体的な税務処理は事業者ごとに事情が異なるため、税理士などの専門家に確認することをおすすめします。

**注意点3:補助事業で収益が出ると返還を求められる場合がある(収益納付)**

制度によっては、補助金を使った事業で一定以上の収益が上がった場合に、補助金額を上限として収益の一部の納付(返還)を求められることがあります。これを「収益納付」と呼びます。すべての補助金にあるわけではありませんが、公募要領に収益納付の定めがあるかどうかは、申請前に確認しておきたいポイントです。

**注意点4:補助金で買ったものは自由に処分できない(財産処分の制限)**

補助金で取得した設備や機械などの財産は、一定期間、勝手に売却・譲渡・廃棄・目的外使用することができません。処分したい場合は事前に公募元の承認が必要で、場合によっては補助金の一部返還を求められます。「使わなくなったから売ろう」が通用しない点は、通常の自己資金での購入と大きく異なるところです。

**注意点5:書類の保存義務が事業終了後も続く**

補助事業に関する帳簿や証拠書類(見積書・請求書・振込記録など)は、事業が終わった後も一定期間(制度により異なりますが、5年程度が一般的)保存する義務があります。期間中は公募元による検査や問い合わせに対応できる状態を保つ必要があります。事業完了後の書類整理・保管のルールを、社内で決めておきましょう。

**注意点6:申請代行の勧誘には慎重に**

補助金の申請支援をうたう業者は数多くありますが、中には高額な成功報酬を請求する業者や、実態と異なる申請内容を作成する悪質な業者も存在します。事業計画の虚偽記載は不正受給にあたり、返還や罰則の対象になるのは申請者自身です。支援を受ける場合は、商工会議所・よろず支援拠点などの公的支援機関や、認定経営革新等支援機関など、信頼できる相手を選びましょう。「必ず採択される」という宣伝文句には特に注意が必要です。

**まとめ**

補助金は事業の後押しになる制度ですが、「課税される」「後からも義務が続く」「もらったお金や買ったものに制約がある」という性質を理解したうえで活用することが大切です。申請前に公募要領をよく読み、不明点は公募元や公的支援機関、税務については税理士などの専門家に確認しましょう。

※本記事は一般的な情報の整理であり、個別の補助金制度の要件・取り扱いはそれぞれ異なります。申請にあたっては、必ず各制度の公式の公募要領をご確認ください。

- 既存のcatch_phrase/summary表示・一次情報源リンク・免責事項表示は維持する

### 完了条件

- 全詳細ページで、`detail`が存在する場合にサニタイズ済みの本文が表示される
- サニタイズにより不正なタグ・スクリプトが除去されていることを確認済み
- スペック表の3項目が、値がある場合のみ正しく表示される
- 既存の一覧・検索・フィルター・ソート機能にデグレがない
- スマホ表示で崩れない

## v2.4 アフィリエイトリンク表示機能(実装待ち)

### 目的

詳細ページ訪問者のうち、対象要件を読み終え関心度が高いユーザーに対して、
関連するIT導入支援サービスを1件だけ提示し、A8.net経由の成果報酬を得る。

### 対象ページ

- 詳細ページ(`templates/detail.html`)のみ
- トップページには設置しない(検索条件確定前のミスマッチを避けるため)

### 表示位置

- 「公式ページを見る」ボタンの直前
- 見出し「[PR] この補助金と相性の良いサービス」+ 小さいボックスを1個のみ表示

### 提携案件(A8.net / すべて株式会社Wiz)

| 案件名 | 対応するキーワード群 | 成果報酬 |
|---|---|---|
| ホームページDX(HPDX) | IT導入・システム・デジタル化系 | 通常30,000円／補助金プラン100,000円 |
| KANBEI SIGN(電子契約) | IT導入・システム・デジタル化系 | 通常7,000円／補助金プラン10,000円 |
| CLOUDPHONE(クラウド電話) | IT導入・システム・デジタル化系 | 通常20,000円／補助金プラン20,000円 |
| URUMAP営業代行 | 小規模事業者・持続化系 | 通常10,000円／補助金プラン20,000円 |

### 出し分けロジック

- 判定対象フィールド:`title`(補助金名)+ `summary`(概要文)の結合テキストに対するキーワード部分一致
- 優先順位(複数マッチ時はこの順で1個のみ採用):
  1. URUMAP営業代行 — キーワード:「小規模事業者」「持続化」(より対象を絞り込んだ具体的な条件のため優先)
  2. ホームページDX(HPDX) — キーワード:「IT導入」「システム」「デジタル化」
  3. KANBEI SIGN・CLOUDPHONE — 同一キーワード群だが、v2.4ではHPDX固定選出。将来のローテーション用に候補として保持のみ(スコープ外参照)
- 修正履歴:当初HPDXを優先していたが、実データ(320件)で検証した結果、小規模事業者・持続化系の
  補助金の多くが同時にデジタル化等のキーワードも含んでおり、URUMAP営業代行が一度も表示されない
  問題が判明したため、優先順位を逆転(2026-07-18、v2.4.1)
- いずれにも該当しない補助金(環境・人材系など) → 何も表示しない

### 表示文言・体裁

- 見出し:「[PR] この補助金と相性の良いサービス」
- リンクには `rel="nofollow sponsored"` を付与
- 新規CSSクラス:`.affiliate-box`, `.affiliate-label`, `.affiliate-link`(既存トーンに合わせた控えめな配色)

### 実装上の注意

- A8.netリンクタグ(実URL)は本SPEC確定時点で未入手。実装ではプレースホルダー(例:`"__PENDING_A8_URL__"`)を用いて構造・ロジック・表示を先行実装し、リンクタグ入手後に値のみ差し替える
- プレースホルダーが残った状態でのpush・本番公開は禁止(完了条件で担保)

- どちらにも該当しない補助金の詳細ページで、ボックスが表示されない
