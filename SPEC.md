## 1. 概要と目的

補助金を探すユーザーが「自分に使える補助金を素早く見つけられる」ことを最優先に、トップページの検索・一覧まわりのUIを刷新する。現状は検索窓の位置・絞り込み軸・一覧の見やすさに課題があり、「調べやすさ」に改善余地がある。今回はサイト全体ではなく **コア体験（トップの検索〜一覧）** に集中する。

---

## 2. ターゲットユーザーと主要な導線

補助金を探している事業者・個人。想定される検索の入口を、ユーザーヒアリングに基づき以下の優先順位で設計する。

| 優先 | 入口 | UIでの扱い |
|------|------|-----------|
| 1 | 地域（都道府県） | 主軸。最も目立たせる |
| 2 | 目的・キーワード | 主軸。チップで直感的に選べるように |
| 3 | 業種 | 絞り込み軸 |
| 4 | 新着 | ソート（並び替え） |
| 5 | 締切が近い順 | ソート（並び替え） |

**主要導線**: トップページ着地 → 検索窓（最上部）で地域・目的・業種を指定 → 一覧をソート（締切/新着）→ 気になる補助金の詳細へ。

## 4. デザイン方針

### トーン
- 信頼感・見やすさ重視。行政/補助金という性質上、誠実で読みやすい印象。
- 情報が探しやすいことを最優先し、装飾より可読性・操作性を優先。

### レイアウト方針
- **検索窓を最上部（ファーストビュー最上部）に大きく配置**。補助金探しが主目的のため。
- **地域（都道府県）を一番目立つ位置に**。47都道府県から選択可能にする。
- **目的・キーワードはチップ形式**。前面によく使う目的を8〜10個ほど表示し、残りは「もっと見る」で展開。
  - ※チップに出す目的カテゴリはおすすめ構成（一般的に検索されやすい目的を想定）。実装時に実データの21キーワードと突き合わせて調整する。
- **業種は絞り込み軸**として、地域・目的の次に配置。
- **新着 / 締切が近い順はソート**として一覧上部に配置。
- **一覧はカード表示**。情報の優先順位を明確に（補助金名・締切・地域・金額を目立たせる）。
- **各カードに2〜3行の要約を常時表示**（どんな補助金か・対象・特徴が一覧上で分かるように）。要約により、詳細ページに飛ばずとも「自分向きか」を一覧で判断できるようにする。
  - ※要約データの扱い：既存データの `summary`（または `catch_phrase`）フィールドを2〜3行に整形して表示する。
  - ※**既存CLAUDE.mdの設計原則「推測でデータを補完しない」に従い、summaryが空／不明の場合はAIによる新規要約生成を行わず、要約欄を非表示にするか「詳細ページでご確認ください」等の定型文で対応する。** データ取得・要約生成ロジックの新設は本フェーズのスコープ外。
- **カードクリック（または「詳細を見る」）で既存の詳細ページへ遷移**（従来型の遷移方式）。詳細ページ自体の改修は今回のスコープ外（遷移先として利用するのみ）。

### 配色・参考URL
- 配色は既存サイトのトーンを踏襲しつつ、可読性を高める方向で調整。

## 5. 技術スタックと制約

- 静的サイト（GitHub Pages ホスティング）
- HTML / CSS / JavaScript（既存構成を踏襲）
- 既存資産: 検索・フィルタ機能（都道府県別・業種別・キーワード）は実装済み。今回はその上に締切/新着ソートを追加し、UI配置・見せ方を刷新する。

- 実装は Claude Code の `/build-site` ワークフローで行う。
- プロジェクトルートの既存 `CLAUDE.md`（技術仕様・設計原則）を踏襲する。特に「一次情報リンク必須」「免責表示」「締切切れの自動除外」「推測でデータを補完しない」の4原則は、今回のUI改善でも遵守する。

## 9. v1.2 追加改善（公開後レビュー 2026-07-14）

フェーズ1実装の公開後、実際の画面を確認したうえでの改善5点。いずれもトップページの範囲内であり、フェーズ1の延長として扱う。

### 9-1. ヒーロー部のサイト説明文の充実
現状の1行説明を、サイトの価値が伝わる2〜3行のリード文に変更する。

採用文面（確定）:
> 全国の補助金・助成金情報を毎日自動収集し、締切順に整理。地域・目的・業種からかんたんに検索できます。すべて公式ページへのリンク付き。

- サイト名「補助金レーダー」は変更しない。
- meta description にも同趣旨の文面を反映する（SEO対策）。

### 9-2. 絞り込み状態のフィードバック表示
チップ・地域・業種・キーワードで絞り込んだ際、一覧の直上に現在の絞り込み条件と件数を表示する。

表示フォーマット(実装時に確定): 条件数によらず統一書式とする。
- 絞り込みなし: 「全299件を表示中」(件数は実データに従う)
- 絞り込みあり(1件でも複数でも): 「〇〇 × △△：N件」(単一条件の場合は「〇〇：N件」、`『』`等の括りは付けない)

- 表示は絞り込み操作のたびに即時更新する。

### 9-3. 絞り込みクリアボタン
「絞り込みをクリア」(すべて表示に戻す)ボタンを設置する。

- 配置は 9-2 のフィードバック表示の隣を想定。
- クリックで地域・目的チップ・業種・キーワードのすべてをリセットし、全件表示に戻す。
- 絞り込みが何もない状態では非表示または非活性とする。

### 9-4. 目的チップの語彙改善
頻度順で選定した結果、補助金サイトでは自明で絞り込みに寄与しない汎用語(「事業」「補助」「支援」「助成」「中小企業」等)がチップに含まれている。これらを除外リストで前面表示から外し、意味のある目的語(「脱炭素」「事業承継」「ものづくり」「人材」等)を優先する。

- 除外リストは generate_site.py 側に定義し、頻度順ロジック自体は維持する。
- 除外した語も「もっと見る」内には表示してよい(検索自体は可能なままとする)。
- ※既存CLAUDE.mdの原則どおり、チップはデータに実在するキーワードのみで構成し、新規語を創作しない。
