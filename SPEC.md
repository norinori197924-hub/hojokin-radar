# SPEC.md — 補助金レーダー(hojokin-radar)トップページUI改善・サイト運用整備 仕様書

**現行バージョン:v1.5(2026-07-14)**
**ステータス:v1.1〜v1.4 実装・デプロイ済み / v1.5 実装待ち**

---

## 1. 概要と目的

補助金レーダーのトップページを、利用者が「自分に使える補助金」を最短で見つけられる検索・一覧体験へ改善する。あわせてアクセス解析(GA4)とAdSense審査に必要な情報開示ページを整備し、公開後の改善判断と収益化準備をデータと運用基盤に基づいて行えるようにする。

## 2. ターゲットユーザーと主要な導線

- 補助金を探す中小企業・個人事業主・自治体関係者
- 主要導線:トップページ → 検索/絞り込み → 一覧カード → 詳細ページ
- 検索の優先順位:地域 → 目的キーワード → 業種 → 新着 → 締切

## 3. 確定仕様(バージョン別)

### v1.1 フェーズ1・コア体験(実装済み)

- 検索窓を最上部に大きく配置
- 3軸フィルター:
  - 地域(47都道府県)
  - 目的キーワードチップ(21語、頻度順)
  - 業種

  - 既存の「締切間近」「新着」2セクションは一覧+ソートに統合

### v1.4 GA4導入(実装済み)

#### 目的

hojokin-radar のアクセス解析のため、GA4トラッキングタグを全ページに導入する。

#### 仕様

- 対象:`generate_site.py` が生成する全HTMLページ(トップページ・詳細ページ含む)
- 挿入位置:`templates/base.html` の既存 `{% if ga4_id %}` ブロックの位置を維持する
  (`<meta charset>` 等の必須メタタグより後、`{% block head_extra %}` の直前)。
  `<meta charset>` より前には配置しない。
- 挿入内容:以下の gtag.js スニペットをそのまま使用する

```html
<!-- Google tag (gtag.js) -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-NKJELNWQKD"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-NKJELNWQKD');
</script>
```

#### 制約・注意

- 測定IDは **G-NKJELNWQKD** のみを使用する(他プロジェクトのID G-1R7H87DFRX を混入させない)
- UI・レイアウト・既存機能への変更は行わない(タグ挿入のみ)
- テンプレートが複数ある場合はすべての生成ページに漏れなく入ること

#### 実装記録

- `scripts/generate_site.py`:`ga4_id = os.environ.get("GA4_ID") or None` を固定値 `ga4_id = "G-NKJELNWQKD"` に変更、未使用となった `import os` を削除(grepで未使用確認済み)
- `templates/base.html`:既存の `{% if ga4_id %}` ブロックをそのまま使用、変更なし
- デプロイ後、GA4「リアルタイム」レポートでアクセス計測を確認済み

#### 完了条件(達成済み)

- 生成された全HTMLの `<head>` 内にスニペットが1回だけ含まれる
- 既存ページの表示・動作にデグレがない
- デプロイ後、GA4「リアルタイム」レポートで自分のアクセスが確認できる

### v1.5 プライバシーポリシー・運営者情報ページ新設(実装待ち)

#### 目的

Google AdSense審査に必要な最低限の情報開示を行うとともに、既存のGA4アクセス解析・将来のAdSense/ASP広告配信について訪問者に明示する。

#### ページ構成

**運営者情報ページ(`about.html`)** ※ファイル名は chosa-rating と統一

- サイト名:補助金レーダー
- 運営者:屋号・ハンドルネームのみ表記(個人名は非公開)
  - 表記例:「運営者:NORI」(chosa-ratingでの表記と統一)
- サイトの目的・運営方針の簡潔な説明
  - 補助金情報の検索・閲覧を支援する個人運営サイトであること
  - 掲載情報は一次情報源(jGrants等)へのリンクを設けていること
  - 情報の正確性について保証するものではない旨(免責の要旨、詳細は既存の免責事項表示を参照)
- 連絡先:メールアドレスを記載 → **[ここに実際のメールアドレスを記入]**
- プライバシーポリシーページへのリンク

**プライバシーポリシーページ(`privacy-policy.html`)**

以下の項目を明記する:

1. **アクセス解析ツールについて**
   - Google Analytics 4(GA4)を導入し、Cookieを使用してアクセス状況を解析していること
   - 収集される情報は匿名の統計データであり、個人を特定するものではないこと
   - Googleのデータ収集・処理について、Googleのプライバシーポリシーへのリンクを設置
2. **広告配信について**
   - 現時点では広告を配信していないが、将来的にGoogle AdSenseおよびアフィリエイト(ASP)広告を導入する可能性があること
   - 導入した場合、第三者配信事業者がCookieを使用し、訪問者の他サイトでのアクセス情報等に基づいて広告を配信する場合があること
   - Googleの広告に関するポリシー(広告設定ページ)へのリンクを設置
3. **Cookieの利用について**
   - 本サイトがCookieを使用する目的(アクセス解析、将来の広告配信)
   - Cookieを無効化する方法(ブラウザ設定)について簡潔に案内
4. **お問い合わせ**
   - 運営者情報ページへの誘導、または連絡先メールアドレスを再掲
5. **改定について**
   - ポリシーは予告なく変更されることがある旨
   - 最終更新日を明記

#### デザイン方針

- 既存サイトのトーン(クリーンな情報サイト)を踏襲
- 新規のビジュアルデザインは不要、既存の `base.html` テンプレート(ヘッダー・フッター含む)をそのまま継承した通常ページとして作成
- 長文になるため、見出し(`<h2>`)で項目を区切り、読みやすさを優先

#### 技術要件

- 静的HTML、`generate_site.py` の生成対象に含める、または `templates/` 配下の静的テンプレートとして直接配置(既存の実装方針に合わせてClaude Codeが実装計画で提案し、承認を得る)
- 全ページ共通のフッターに「プライバシーポリシー」「運営者情報」へのリンクを常時表示
  - `templates/base.html` のフッター部分に追記
  - リンクは既存のスタイルシート参照と同様に `{{ base_path }}` を用いた相対パスで記述し、トップページ・詳細ページ(`s/{id}.html`)のどちらからもリンク切れしないこと
- `sitemap.xml` が存在する場合は、新設2ページのURLを追加する
- ページ本文(プライバシーポリシー・運営者情報の文面)は、実装計画の承認後・コード実装前にドラフトを提示して確認を得ること
- 既存の検索・フィルター・ソート等のUI機能には一切手を入れない

#### 完了条件

- `privacy-policy.html`、`about.html` が生成され、正しく表示される
- 全ページのフッターから両ページにアクセスできる(トップページ・詳細ページの両方からリンク切れがないこと)
- プライバシーポリシーに GA4・将来の広告配信・Cookieについての記載が明記されている
- 運営者情報ページに連絡先(メールアドレス)が記載されている
- `sitemap.xml` が存在する場合、新設2ページのURLが含まれている
- 既存ページの表示・動作にデグレがない
- スマホ表示で崩れない

## 4. デザイン方針(全体)

- トーン:信頼感のあるクリーンな情報サイト
- 配色:ブルー基調(ヒーローに薄いブルーグラデーション)、バッジ配色はWCAG AAコントラスト比を満たすこと
- フォント:既存のまま変更しない

## 5. 技術スタックと制約(全体)

- 静的サイト:`generate_site.py` によるHTML生成
- ホスティング:GitHub Pages(`docs/` 配下、`.nojekyll` 設置済み)
- データソース:jGrants API(詳細はCLAUDE.md参照)
- デプロイ:git push → GitHub Actions
- リモートの日次自動データ更新あり:push前に `git fetch` → コンフリクト確認 → (あれば)マージ → `generate_site.py` 再実行 → コミット → push の手順を厳守
- コミットメッセージは `git commit -F`(ファイル経由)方式を使用
