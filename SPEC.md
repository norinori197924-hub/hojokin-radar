# SPEC.md — 補助金レーダー(hojokin-radar)トップページUI改善 仕様書

**現行バージョン:v1.4(2026-07-14)**
**ステータス:v1.1〜v1.3 実装・デプロイ済み / v1.4 実装待ち**

---

## 1. 概要と目的

補助金レーダーのトップページを、利用者が「自分に使える補助金」を最短で見つけられる検索・一覧体験へ改善する。あわせてアクセス解析(GA4)を導入し、公開後の改善判断をデータに基づいて行えるようにする。

## 2. ターゲットユーザーと主要な導線

- 補助金を探す中小企業・個人事業主・自治体関係者
- 主要導線:トップページ → 検索/絞り込み → 一覧カード → 詳細ページ
- 検索の優先順位:地域 → 目的キーワード → 業種 → 新着 → 締切

## 3. 確定仕様(バージョン別)

### v1.1 フェーズ1・コア体験(実装済み)

- 検索窓を最上部に大きく配置
- 3軸フィルター:
  - 地域(47都道府県)
  - 目的キーワードチップ(21語、頻度順)
  - 業種
- ソート機能:「締切が近い順」「新着順」
  - 既存の「締切間近」「新着」2セクションは一覧+ソートに統合
  - 新着はNEWバッジで表示
- 一覧カードに2〜3行の要約を表示
  - 既存 `catch_phrase` / `summary` フィールドを使用
  - **AIによる新規生成はしない**(フィールドが空の場合は非表示)
- カードクリックで詳細ページへ遷移(詳細ページ自体の改修はスコープ外)

### v1.2 公開後レビュー第1回(実装済み)

- ヒーローにサイト説明のリード文を追加
  - meta description / og:description にも反映
- 絞り込み条件と件数のフィードバック表示(統一フォーマット)
- 「絞り込みをクリア」ボタンを追加
- 目的チップの汎用語(「事業」「補助」等)を除外リストで前面から除外

### v1.3 公開後レビュー第2回(実装済み)

- ヒーローを3段構成に整理:
  1. サイト名
  2. キャッチコピー「あなたに使える補助金を見つける」
  3. リード文
  - 旧説明文は削除
- 広告枠の配置を2箇所に限定:
  - 検索エリア下
  - 一覧最下部
  - 一覧見出し直下の広告枠は削除
  - ※ソート機能とのデグレ回避のため「5件目の後」ではなく最下部を採用
- デザイン改善案A:
  - ヒーロー背景に薄いブルーグラデーション
  - カードのホバー時浮き上がり効果
  - 締切/NEWバッジの配色強化(WCAG AAコントラスト比確認済み)
- チップの挙動・見た目、UIレイアウト構造、フォントは変更なし

### v1.4 GA4導入(実装待ち)

#### 目的

hojokin-radar のアクセス解析のため、GA4トラッキングタグを全ページに導入する。

#### 仕様

- 対象:`generate_site.py` が生成する全HTMLページ(トップページ・詳細ページ含む)
- 挿入位置: `templates/base.html` の既存 `{% if ga4_id %}` ブロックの位置を維持する(`<meta charset>` 等の必須メタタグより後、`{% block head_extra %}` の直前)。`<meta charset>` より前には配置しない
- 挿入内容:以下の gtag.js スニペットをそのまま使用する

```html
<!-- Google tag (gtag.js) -->
<script async src="https://www.googletagmanager.com/gtag/js?id=G-NKJELNWQKD"></script>
<script>
  window.dataLayer = window.dataLayer || [];
  function gtag(){dataLayer.push(arguments);}
  gtag('js', new Date());
  gtag('config', 'G-NKJELNWQKD');
</script>
```

#### 制約・注意

- 測定IDは **G-NKJELNWQKD** のみを使用する(他プロジェクトのID G-1R7H87DFRX を混入させない)
- UI・レイアウト・既存機能への変更は行わない(タグ挿入のみ)
- テンプレートが複数ある場合はすべての生成ページに漏れなく入ること

#### 完了条件

- 生成された全HTMLの `<head>` 内に上記スニペットが1回だけ含まれる
- 既存ページの表示・動作にデグレがない
- デプロイ後、GA4「リアルタイム」レポートで自分のアクセスが確認できる

## 4. デザイン方針

- トーン:信頼感のあるクリーンな情報サイト
- 配色:ブルー基調(ヒーローに薄いブルーグラデーション)、バッジ配色はWCAG AAコントラスト比を満たすこと
- フォント:既存のまま変更しない

## 5. 技術スタックと制約

- 静的サイト:`generate_site.py` によるHTML生成
- ホスティング:GitHub Pages(`docs/` 配下、`.nojekyll` 設置済み)
- データソース:jGrants API(詳細はCLAUDE.md参照)
- デプロイ:git push → GitHub Actions
- リモートの日次自動データ更新あり:push前に `git fetch` → コンフリクト確認 → (あれば)マージ → `generate_site.py` 再実行 → コミット → push の手順を厳守

## 6. スコープ外のこと

- 詳細ページの再設計(フェーズ2で対応)
- 市区町村レベルの検索(フェーズ2で対応)
- プライバシーポリシー・運営者情報ページの新設(AdSense審査準備フェーズで対応)
- 実際の広告タグの挿入(上記ページ新設後)
- AIによる要約・データの新規生成(全フェーズ共通の禁止事項)
- 期限切れ補助金の表示(自動除外を維持)

## 7. 完了条件(全体)

- 全ページがスマホ表示で崩れない
- 検索・絞り込み・ソートが正しく動作する
- カードから詳細ページへの遷移が正しく動作する
- 一次情報源へのリンクが維持されている
- 免責事項の表示が維持されている
