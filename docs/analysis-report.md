# 分析レポート（広告枠プレースホルダー追加）

担当: analyzer サブエージェント（パフォーマンス・SEO・GA4観点）

## パフォーマンス

| 指摘 | 重大度 | 対応 |
|---|---|---|
| 現時点(プレースホルダーのみ)ではCLS・追加リクエストなし、min-height:100pxで最低限確保済み | -（問題なし） | 対応不要 |
| min-height:100pxは「ゼロ回避」の暫定値。実際のAdSense広告フォーマット(150〜300px超になりうる)確定時に差し替えないとCLSが発生しうる | 高 | 実広告差し込み時に対応（今回は設計として明記） |
| 広告枠自体はJS不要な静的HTML/CSSのみ、render-blockingリソース増加なし | -（問題なし） | 対応不要 |

## SEO

| 指摘 | 重大度 | 対応 |
|---|---|---|
| メタタグ・見出し構造(h1/h2)への影響なし | -（問題なし） | 対応不要 |
| `ad-slot-index-top`がページ本文冒頭(h2「締切間近」より前)に配置されており、実広告差し込み後にAds Above the Foldガイドライン(ファーストビュー過度占有)に抵触しないか要確認 | 中 | 実広告差し込み時に確認（今回はプレースホルダーの見た目上問題なし） |
| index.htmlにh1が存在しない（既存の問題、今回のスコープ外） | -（既存課題） | 対応不要 |
| detail.htmlの`ad-slot-detail-top`はパンくず直後・本文(h1・table)より前だが、パンくずはナビゲーション要素でありSEO上の重大な問題はない | -（問題なし） | 対応不要 |

## 計測タグ

| 指摘 | 重大度 | 対応 |
|---|---|---|
| GA4スクリプト(`<head>`内)と広告枠(`<body>`内)は完全に独立、重複・欠落なし | -（問題なし） | 対応不要 |
| 広告インプレッション/クリック計測のフックが現状ない(スコープ通り) | 低 | 今回は見送り（将来AdSense導入時に`id`属性を流用してGA4連携を検討） |

## 今回対応する項目

なし（高重大度の指摘は「実際の広告フォーマット確定時に再設定する」性質のもので、
プレースホルダー段階の今回は対応不要と判断。design.mdに申し送り事項として明記する）

## 今回スコープ外（将来の申し送り事項）

1. 実広告タグ差し込み時に、実際のフォーマットに合わせて`.ad-slot`の`min-height`
   （必要ならレスポンシブ対応も）を再設定する
2. `ad-slot-index-top`のファーストビュー占有について、実広告差し込み後に
   Ads Above the Foldガイドラインの観点で再確認する
3. 広告インプレッション/クリックのGA4計測を導入する場合、`id`属性
   （`ad-slot-xxx`）をイベントlabelとして流用する設計にできる
